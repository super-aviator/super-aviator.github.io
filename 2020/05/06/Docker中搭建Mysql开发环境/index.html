Docker中搭建Mysql开发环境

今天上班划水一天，尝试在Docker中搭建mysql开发环境，记录一下。

首先需要查询镜像，然后pull到本地，具体的步骤就不说了，非常简单。

拉取进行之后，使用命令docker images -a 可以看到下载到本地的mysql镜像：

拉取之后，首先我们需要在主机中创建mysql的配置文件/etc/mysql/my.cnf，其中的datadir指定存放文件的目录：

这里使用mysql默认的配置，存放数据的文件夹路径为/var/lib/mysql；然后使用mkdir /data/mysql/data命令在主机上创建一个用于存放mysql数据的文件夹，启动时挂载到容器的/var/lib/mysql目录下，确保数据存放在主机中而非容器中。

完成这些，就可以启动docker容器了，使用命令：

docker run -itd --name mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD="123456" -v /etc/mysql/my.cnf:/etc/mysql/my.cnf -v /data/mysql/data:/var/lib/mysql  mysql:8
  • -e 通过指定环境变量MYSQL_ROOT_PASSWORD="123456"指定root用户的密码为123456
  • -v /etc/mysql/my.cnf:/etc/mysql/my.cnf 挂载配置文件
  • -v /data/mysql/data:/var/lib/mysql 挂载存放数据的文件夹

注意，使用此命令启动mysql时，命令的最后不能指定/bin/bash路径，否则mysql会无法启动。并且应该确保存放数据的文件夹为空，否则mysql也会启动失败

执行上面的命令后，使用docker top mysql查看mysql容器中的mysqld进程是否启动。然后可以通过命令ls /data/mysql/data查看存放数据的文件夹是否有数据：
