Java 8

Java8是继Java5更新以来对Java影响最大的一次更新，包括支持函数式编程、Lambda等，使得Java代码不再像老太太的裹脚布那样又臭又长！

Lambda

Java8中新增了Lambda表达式，Lambda表达式由参数、箭头以及主体构成，例如：

值得注意的是，Lambda表达式的主体如果不带大括号{}，此时如果主体是一个表达式，则默认带return显示返回语句，如果主体是一个语句，则表示返回Void。如果带了大括号并且需要返回，则必须显示的使用return关键字，例如：

  • (Integer i)->{"熊乾坤";}：此Lambda中的主体是一个表达式而不是语句，所以需要加上return。
  • (Integer)->return "熊乾坤";：此Lambda的主体是一个控制流语句，要使此Lambda有效，则需要使用花括号。

方法引用

Java中一等值指的是可以对其进行操作的类型，例如数值类型、引用等，一等值可以进行操作或者作为方法传递，而其他用于描述或者处理一等值的类型成为二等值，例如方法、类、接口等，Java8中将方法升级成为了一等值，即方法也可以赋值给一个引用，或者作为方法的参数进行传递，这称为方法引用，这也是Lambda使用的基础条件，方法引用使用类名加上::加上方法名构成，例如：

默认方法

Java8中为接口增加了大量的方法，例如Collection接口的filter()、List接口的sort()等方法，此时如果直接将方法声明为抽象方法，则需要Collection接口的所有实现类全部实现该方法，最糟糕的是如果开发者自己实现了Collection接口，则这些代码使用JDK1.8则无法通过编译，必须加上这些新增的方法的实现，这样从而导致大量的工作。
